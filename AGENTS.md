## Instructions for Code Edits

When making code edits, please follow these guidelines to ensure consistency and maintainability across the cerberus codebase:

- The cerberus repository has code multiple directories, after making edits make sure to fix all locations as needed:
    - src/
    - tests/
    - notebooks/
    - docs/
    - examples/
- Whenever writing new code, write appropriate tests in the tests/ directory. Cover, to catch future regressions:
    - API
    - Edge cases
    - Logic
- At the end of your edits, update docs/

## Instructions for Behavior

- Before editing files always ask for explicit permission.
- To get permission, explain what you want to change and why.
- Evaluate as much of the codebase as possible to find all relevant locations for your edits.
